Platform Architecture

Mondal's E-Commerce engine is an advanced Laravel 11 application structured explicitly for multi-tenancy (Multi-Vendor). The platform operates on isolated domains separating Customers, Vendors, and Administrators while housing dynamically swappable Themes and Plugins.

Storefront Routing

Guest checkouts, session-based cart memory, vendor directories (/stores/{slug}), dynamic categories, and variant pricing algorithms.

Admin Dashboard

Strict pipeline controls for orders, plugin zipper tools, theme uploading tools, payouts dispatch logic, and user permissions.

Vendor Control

Independent catalog management, earning calculations, and localized fulfillment handling within the vendor.* middleware scopes.

Complete Directory Structure

Every element respects SOLID principles. Instead of polluting web.php or the base Controllers folder, elements are abstracted into subdirectories.

Database Schema & Entities

A few crucial tables power the integrity of the ecosystem. Notice the strong emphasis on exact Data Types.

products table

Keys: id, vendor_id, brand_id, category_id
Attributes: price, compare_price, quantity, low_stock_threshold, is_featured, track_quantity
Status (Enum): draft, pending, approved, rejected

orders table (STRICT ENUMS)

To assure financial tracking integrity, you CANNOT overwrite order status with random strings. The API/Admin updates rigidly test against these constraints:

Col: status (Fulfillment State)
  • pending (Awaiting merchant action)
  • confirmed
  • processing
  • shipped
  • delivered
  • completed
  • cancelled
Col: payment_status (Finance Line)
  • pending (Default on instantiation)
  • paid (Validated transaction)
  • partially_paid
  • refunded
  • failed (Gateway rejection)

Custom Extensibility 1: Theme Manager

A dedicated service to swap frontend skins seamlessly without code deployments.

Featured: Lovemad Premium Theme

The lovemad theme is the new premium default, offering high-conversion layouts, glassmorphic UI elements, and full mobile responsiveness.

  1. Dynamic Namespacing: Use View::prependNamespace() to force Blade to look in the active theme folder first.
  2. Static Asset Symlinking: Automatic copy/link from resources/themes/assets/ to public/themes/.
  3. Isolated Configurations: Per-theme customization via ThemeController@customize.
<!-- Mandatory Asset Call for themes --> <link rel="stylesheet" href="{{ @themeAsset('css/main.css') }}">

Custom Extensibility 2: Plugin Engine

Allows deep functional override via ZIP uploads (e.g., Payment Gateways, ERP Sync). This uses the app('plugin.manager') container singleton.

Plugin Pipeline & Admin Hooks

Installation: ZIP uncompressed ➡️ Provider registration ➡️ Migration execution.

Discovery: Automatic detection of views/settings.blade.php for auto-generated config windows.

Purge: Total destruction of plugin records and physical disk directories to prevent bloat.

Custom Extensibility 3: Fallback Image Engine

The HasFallbackImage trait ensures no broken images ever appear on the storefront. It utilizes deterministic seeds to provide consistent colors for specific products or users.

// Implementation Example in Blade <img src="{{ $product->getFallbackImage($product->image, $product->name, '600x600') }}" /> // Avatar specific call <img src="{{ $user->getFallbackImage($user->avatar, $user->name, '100x100', 'avatar') }}" />

Technical Stack: External APIs (UI Avatars, Placehold.co) with local existence checks on the public disk.

CI/CD & Automated Deployment

Mondal's Engine features a dual-stack deployment strategy for maximum reliability.

GitHub Actions Pipeline

Automated testing and server synchronization via SSH / Cloudflare Tunnels. Files are pushed directly to production upon successful tests.

Docker One-Click (`deploy.sh`)

Orchestrates container builds, MySQL health checks, Laravel cache warming, and permission hardening in a single command.
