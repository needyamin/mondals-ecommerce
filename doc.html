Platform Architecture

Mondal's E-Commerce engine is an advanced Laravel 11 application structured explicitly for multi-tenancy (Multi-Vendor). The platform operates on isolated domains separating Customers, Vendors, and Administrators while housing dynamically swappable Themes and Plugins.

Storefront Routing

Guest checkouts, session-based cart memory, vendor directories (/stores/{slug}), dynamic categories, and variant pricing algorithms.

Admin Dashboard

Strict pipeline controls for orders, plugin zipper tools, theme uploading tools, payouts dispatch logic, and user permissions.

Vendor Control

Independent catalog management, earning calculations, and localized fulfillment handling within the vendor.* middleware scopes.

To assure financial tracking integrity, you CANNOT overwrite order status with random strings. The API/Admin updates rigidly test against these constraints:

  • pending (Default on instantiation)

  • refunded
  • failed (Gateway rejection)

Custom Extensibility 1: Theme Manager

A dedicated service to swap frontend skins seamlessly without code deployments.

  1. Dynamic Namespacing: The ThemeServiceProvider boots on startup, fetches the active theme name from the DB (e.g. "sunset"). It intercepts Laravel's View Engine and tells it to search in resources/themes/sunset/views/ before searching in resources/views.
  2. Static Asset Symlinking: Because raw folders outside of public/ can't be served, the Manager uses File::copyDirectory() [on Windows] or File::link() [on Posix] to push resources/themes/sunset/assets/ to public/themes/sunset/.
  3. Isolated Configurations: Themes have a theme.json file storing configurable inputs (colors, toggles). The ThemeManager::getCustomization() allows DB saved overrides specifically per-theme.
<!-- Mandatory Asset Call for themes --> <link rel="stylesheet" href="{{ @themeAsset('css/main.css') }}">

Custom Extensibility 2: Plugin Engine

Allows deep functional override via ZIP uploads (e.g., Payment Gateways, ERP Sync). This uses the app('plugin.manager') container singleton.

Plugin Pipeline & Admin Hooks

1. The Admin uploads AcmeGateway.zip via Admin\PluginController@upload

2. ZIP is uncompressed into base_path('/plugins').

3. Admin clicks 'Install'. plugin.manager->activate($plugin) registers the Plugin Provider, executing Database Migrations contained inside the plugin.

4. If the plugin has a file mapping to plugins/{slug}/views/settings.blade.php, the Admin Panel automatically displays a "Settings" configuration window for it!

5. Purge Mechanism: Admins can now 'Purge from Disk' to completely delete a plugin's physical folder, ideal for cleaning up duplicates.

Payment & Shipping Hooks

bKash Integration: Supports Tokenized Checkout (v1.2.0-beta) using X-APP-Key and Authorization: Bearer headers. Logic is isolated in BkashGateway.php.

Unified Shipping: The CheckoutController merges DB-based methods with plugin-hooked methods into a single availShipping collection, allowing plugins like Pathao to inject rates dynamically.
